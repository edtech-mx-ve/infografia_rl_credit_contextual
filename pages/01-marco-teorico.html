1.1 Machine Learning y aprendizaje por refuerzo

El aprendizaje por refuerzo es uno de los paradigmas de Machine Learning. Su interés principal no es solamente predecir un resultado, sino aprender una política de decisión.

1.2 ¿Por qué no se utiliza un MDP secuencial?

Un Proceso de Decisión de Markov completo requiere múltiples estados y transiciones. El dataset de esta práctica no contiene trayectorias temporales del mismo cliente.

estado actual

→ recompensa
→ nuevo estado
→ nueva acción
Formulación adecuada: una solicitud → un contexto → una acción → una recompensa → fin del episodio.

1.4 LinUCB

LinUCB combina una estimación de recompensa con una bonificación por incertidumbre.

θₐ = Aₐ⁻¹ bₐ
pₐ = θₐᵀx + α √(xᵀAₐ⁻¹x)
Explotación
θₐᵀx

Utiliza lo aprendido para estimar la recompensa.

Exploración
α · incertidumbre

Favorece acciones menos conocidas cuando existe incertidumbre.

Principio central: explotación + exploración = decisión contextual.

Papel de alpha

alpha pequeño → mayor explotación
alpha grande  → mayor exploración

Valores evaluados: 0.1, 0.5, 1.0 y 2.0.

En la ejecución registrada, los cuatro valores produjeron las mismas métricas de validación. Por eso alpha = 0.1 fue seleccionado por desempate de orden, no porque fuera significativamente mejor.
